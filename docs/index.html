ControllerSets

Express Controller Sets

An advanced, unified toolkit for Express.js that accelerates API development by providing automated CRUD operations, dynamic routing capabilities, and robust Amazon S3 file upload handling out of the box.

Node 20+ Mongoose 9+ Express 5+ Multer 2+

Designed to help you build production-ready APIs faster by automating repetitive controller logic and complex S3 middleware configuration while maintaining type safety and maximum flexibility.

Installation

Install the package using your favorite package manager:

npm install express-controller-sets mongoose express multer multer-s3 dotenv @aws-sdk/client-s3

Environment Variables

Configure your application and S3 credentials via environment variables:

NODE_ENV=development
S3_ENDPOINT=your-endpoint.com
S3_SPACES_KEY=your-key
S3_SPACES_SECRET=your-secret
S3_BUCKET_NAME=your-bucket
S3_REGION=us-east-1

Global Error Handling

To ensure all errors (including database validation failures) are returned in professional JSON format, use the built-in errorHandler middleware.

import express from 'express';
import { createRouter, errorHandler } from 'express-controller-sets';

const app = express();
app.use(express.json());

// ... your routes ...

// MUST BE ADDED LAST: Global JSON Error Handler
app.use(errorHandler);

Standardized Error Response

All errors caught by this middleware will return a consistent 400 or 500 status code with the following body:

{
    "success": false,
    "error": "Detailed error message here",
    "stack": "... (only visible in development) ..."
}

Quick Start

Build a full-featured API for your model in just a few lines of code.

import express from 'express';
import { createRouter } from 'express-controller-sets';
import Product from './models/Product.js';

const app = express();
app.use(express.json());

// Create all CRUD routes automatically
const productRouter = createRouter({
    model: Product,
    orderBy: '-createdAt', // Sort by newest
    search: ['name', 'category.name'], // Enable ?search= or ?s= for multi-field search (including relational)
    query: ['category']    // Enable ?category= filtering
});

app.use('/api/products', productRouter);

Controller Sets

The ControllerSets class provides the core logic for handling Mongoose operations. You can use it directly for custom route handling by instantiating it with specific options.

Constructor Parameters

Parameter Type Description
model Mongoose Model Required. The Mongoose model that the controller will operate on (e.g., Product, User).
orderBy String The default sorting behavior when fetching lists (e.g., '-createdAt' for newest first). Defaults to 'none'.
query Array<String> An array of field names that are allowed to be filtered directly via query parameters (e.g., ['category', 'status'] allows ?status=active).
search Array<String> | String Fields to include when a user provides the ?search= or ?s= query parameter. Supports relational dot notation (e.g., 'category.name').
runAfterCreate Function An optional asynchronous callback function executed immediately after a new document is created. It receives the newly created document as its argument.
onGet Function An optional hook (req, res) that returns { populates, selects } to conditionally modify how data is populated or selected on all GET operations. 
import { ControllerSets } from 'express-controller-sets';
import Product from '../models/Product.js';

const productController = new ControllerSets(
    Product,             // 1. model
    '-createdAt',        // 2. orderBy
    ['category'],        // 3. query (Filterable fields)
    ['name', 'tags'],    // 4. search (Searchable fields)
    async (doc) => {},   // 5. runAfterCreate callback
    (req, res) => {      // 6. onGet hook
        return { selects: '-hiddenField' }; 
    }
);

// Use in manual routes
router.get('/', productController.getAll);
router.post('/', productController.create);
router.get('/:id', productController.getById);
router.patch('/:id', productController.update);
router.delete('/:id', productController.delete);

S3 File Upload Middleware

Handle file uploads to S3-compatible storage with ease. The middleware automatically updates req.body with S3 locations.

Type: This is a specialized Express middleware wrapper around Multer and the AWS SDK. It intercepts multipart form data, optionally compresses images using Sharp, securely uploads them directly to S3 via memory buffers, and seamlessly forwards execution back to your route handler.

Middleware Arguments

Argument Type Description
req Object The standard Express Request object.
res Object The standard Express Response object.
next Function The standard Express Next callback.
uploadPath String The destination folder path inside your S3 bucket (e.g., 'uploads/').
fields Array Multer field configurations specifying expected files (e.g., [{ name: 'avatar', maxCount: 1 }]).
imgOptimizationsDefault String (Optional) The fallback image compression level ('low', 'medium', 'high'). 
import { fileUploadMiddleware } from 'express-controller-sets';

router.post('/upload', (req, res, next) => {
    // Pass the optimization level (e.g., 'medium') as the 6th parameter
    fileUploadMiddleware(req, res, next, 'uploads/', [
        { name: 'avatar', maxCount: 1 }
    ], 'medium');
}, (req, res) => {
    res.json({ url: req.body.avatar });
});

Image Compression & Optimization

Drastically reduce your S3 storage costs and improve API response times by automatically optimizing uploaded images via the imgOptimizations feature. Powered by Sharp, the module utilizes a highly efficient, multi-pass binary search algorithm to dynamically scale down the file size while preserving original image dimensions and visual fidelity.

Available Compression Profiles:

Level Target File Size Best Used For
low 75% - 80% of original size High-fidelity images where maximum quality is prioritized over bandwidth savings.
medium / med 60% - 65% of original size General purpose uploads (e.g., product photos, hero images) offering the perfect balance of quality and size.
high 40% - 45% of original size Avatars, thumbnails, and heavy traffic endpoints where aggressive bandwidth savings are crucial.

You can seamlessly integrate image optimization into your workflow in two ways:

1. Route-Level Setup (Static Default)

Specify the default compression level when initializing your router:

import { createRouterS3upload } from 'express-controller-sets';

const router = createRouterS3upload({
    model: User,
    path: 'avatars/',
    fields: [{ name: 'avatar', maxCount: 1 }],
    imgOptimizations: 'medium' // All avatars will be compressed to 60-65% size by default
});

2. Client-Controlled (Dynamic Override)

Clients can dynamically specify or override the optimization level per-request by passing the imgOptimizations parameter. This parameter can be passed in the query string, multipart form-data fields, or headers (imgOptimizations or x-img-optimizations):

// Dynamic query parameter override
fetch('/api/users?imgOptimizations=high', {
    method: 'POST',
    body: formData
});

3. Bypassing / Ignoring Optimizations

If you have set a route-level default (e.g., 'medium') but want to allow certain uploads to bypass compression entirely, clients can explicitly pass none or false as the imgOptimizations parameter. This guarantees the original file is uploaded untouched:

// Ignore optimizations for this specific upload
fetch('/api/users?imgOptimizations=none', {
    method: 'POST',
    body: formData
});

Dynamic Routers

Accelerate your API development by utilizing our dynamic routing helper functions. With a single function call, you can instantly scaffold full-featured Express routers equipped with comprehensive CRUD operations, integrated S3 file uploads, and advanced querying capabilities out-of-the-box.

Basic Router

import { createRouter } from 'express-controller-sets';
import Product from './models/Product.js';

const router = createRouter({
    model: Product,
    orderBy: '-createdAt',              // Sort order for GET requests
    query: ['category', 'status'],      // Allowed query parameters for filtering
    search: ['name', 'description'],    // Fields to search when ?search=term is used
    runAfterCreate: async (doc) => {},  // Hook executed after document creation
    onGet: (req, res) => { return {} }, // Hook to conditionally apply populate/select
    middlewares: [authMiddleware]       // Array of middlewares to run before controllers
});

Router with S3 File Upload

Automatically combines CRUD logic with S3 file upload middleware.

import { createRouterS3upload } from 'express-controller-sets';
import User from './models/User.js';

const router = createRouterS3upload({
    model: User,
    orderBy: '-createdAt',              // Sort order
    query: ['role'],                    // Allowed query params
    search: ['username', 'email'],      // Searchable fields
    runAfterCreate: async (doc) => {},  // Post-create hook
    onGet: (req, res) => { return {} }, // Conditional populate/select
    middlewares: [authMiddleware],      // Route middlewares
    path: 'avatars/',                   // S3 folder path for uploads
    fields: [{ name: 'avatar', maxCount: 1 }], // Multer upload fields
    imgOptimizations: 'medium'          // Optional image compression default
});

Registering the Router

Because the helper functions return a standard Express router, you register them in your main application exactly as you would normally:

import express from 'express';
// import productRouter from './routes/productRouter.js'

const app = express();

app.use(express.json());
app.use('/api/products', productRouter);

Conditional Populate (onGet)

The onGet hook gives you request-time control over .populate() and .select() for all GET operations (getAll, getById, and paginated results). It is entirely optional — without it, ControllerSets works exactly as before.

Hook Signature

The hook receives (req, res) and returns an object with populates and/or selects:

onGet(req, res) {
    return {
        populates,  // string | object | array — passed to .populate()
        selects,    // string — passed to .select()
    };
}

Simple Demo

Use simple query parameters to conditionally include or exclude fields.

import { createRouter } from 'express-controller-sets';
import Product from './models/Product.js';

const productRouter = createRouter({
    model: Product,
    query: ['category'],
    onGet: (req, res) => {
        // If ?noUsers=true is passed, exclude the 'users' field
        if (req.query.noUsers) {
            return { selects: "-users" };
        }
        // Otherwise, populate the users field
        return {
            populates: [{ path: "users.user", select: "name email" }]
        };
    }
});

Complex Demo

Chain multiple populates and make decisions based on both query parameters and user roles.

const orderRouter = createRouter({
    model: Order,
    query: ['status'],
    onGet: (req, res) => {
        const populates = [{ path: "customer", select: "name email" }];
        let selects = "";

        // Add detailed item data if requested
        if (req.query.detailed === "true") {
            populates.push({ path: "items.product", select: "name price image" });
        } else {
            selects = "-items";
        }

        // Only admins can see who the order is assigned to
        if (req.user?.role === "admin") {
            populates.push({ path: "assignedTo" });
        }

        return { populates, selects };
    }
});

Advanced Demo (Modular Reuse)

Extract your onGet logic into isolated, reusable hook functions to share complex population logic across multiple resource routers.

// hooks/populateUsers.js
export const populateUsers = (req, res) => {
    if (req.query.noUsers) return { selects: "-users" };
    return { populates: [{ path: "users.user", select: "name email" }] };
};

// routes/products.js
import { populateUsers } from '../hooks/populateUsers.js';
const productRouter = createRouter({ model: Product, onGet: populateUsers });

// routes/orders.js — same hook, zero duplication
const orderRouter = createRouter({ model: Order, onGet: populateUsers });

TypeScript Support

The package comes with built-in TypeScript definitions. You can pass your Mongoose document interfaces to strongly type the controllers and routers.

import express from 'express';
import { createRouter } from 'express-controller-sets';
import mongoose, { Document } from 'mongoose';

// 1. Define your interface
interface IUser extends Document {
    name: string;
    email: string;
    age: number;
}

// 2. Define or import your model
const UserModel = mongoose.model<IUser>('User', new mongoose.Schema({
    name: String,
    email: String,
    age: Number
}));

// 3. Create a strongly-typed router
const userRouter = createRouter<IUser>({
    model: UserModel,
    orderBy: 'name',
    search: ['name', 'email']
});

const app = express();
app.use('/api/users', userRouter);

API Reference

Option Type Description
model Mongoose Model Required model for database operations.
orderBy String Default sorting field (e.g. '-createdAt').
query Array<String> Fields allowed for automatic query filtering.
search Array<String> | String Array of field names (or a single string) for regex searching. Supports relation dot notation (e.g. category.name).
runAfterCreate Function Optional callback executed after successful create.
middlewares Array<Function> Array of middlewares applied to all routes.
imgOptimizations String Optional default image optimization preset for S3 uploads: 'low', 'medium', 'med', or 'high'.
onGet Function Optional hook (req, res) => { populates, selects } for conditional .populate() and .select() on all GET queries. Entirely optional.

Note on Pagination

Pagination is supported out-of-the-box for getAll queries. By appending ?page=1&pageSize=50, the response automatically provides a pagination metadata object alongside your results.

API Endpoints

All routers created with createRouter or createRouterS3upload follow standard RESTful patterns and include a powerful built-in filtering engine.

Method Endpoint Description
GET / List records with support for pagination, search, and custom filters.
POST / Create a new record. Supports multipart file uploads on S3 routers.
GET /:id Retrieve a single record by its 24-character hex ID.
PATCH /:id Update an existing record partially. Supports file updates on S3 routers.
DELETE /:id Permanently remove a record from the database.

Query Parameters (Filtering Engine)

The GET / endpoint supports several parameters out-of-the-box to filter and paginate your data.

Parameter Description Example
page The page number (starts at 1). ?page=2
pageSize Number of items per page (default: 10, max: 100). ?pageSize=50
search / s Global search across configured fields. ?search=laptop
sort Field to sort by. Prefix with - for descending. ?sort=-price
[fieldName] Direct equality filter for any configured query field. ?category=65a...
rangeField The numeric field name to apply a range filter on. Works together with range parameter. ?rangeField=price
range The range boundaries in format min-max. Supports min-only (min-) or max-only (-max) ranges. ?range=10-100
compareField The field name to perform a dynamic comparison on (e.g., price, age, etc.). ?compareField=price
compareValue The target value to compare against. Tries to parse as a number if valid, otherwise treats as a string. ?compareValue=50
compareOperator The comparison operator: gt (>), gte (>=), lt (<), lte (<=), ne (!=), eq (==). Default is eq. ?compareOperator=gt

Usage Examples

1. Fetching Paginated & Sorted Results

// Fetch the second page of active products, sorted by price
fetch('/api/products?page=2&pageSize=20&sort=-price&status=active')
    .then(res => res.json())
    .then(data => {
        console.log(data.data); // Array of items
        console.log(data.pagination); // { total, page, pageSize, totalPage }
    });

2. Filtering by Numeric Range

Filter records dynamically by a numeric range using rangeField and range query parameters.

// Fetch products priced between $10 and $100
fetch('/api/products?rangeField=price&range=10-100')
    .then(res => res.json())
    .then(data => console.log(data.data));

// Fetch users aged 18 or older (min only)
fetch('/api/users?rangeField=age&range=18-')
    .then(res => res.json())
    .then(data => console.log(data.data));

3. Dynamic Field Comparison

Filter records dynamically on a field using comparison operators like greater than (gt), greater than or equal (gte), less than (lt), less than or equal (lte), not equal (ne), or equal (eq).

// Fetch products with price greater than 50
fetch('/api/products?compareField=price&compareValue=50&compareOperator=gt')
    .then(res => res.json())
    .then(data => console.log(data.data));

// Fetch active users with age less than or equal to 30
fetch('/api/users?status=active&compareField=age&compareValue=30&compareOperator=lte')
    .then(res => res.json())
    .then(data => console.log(data.data));

4. Creating a Record with File Upload

When using createRouterS3upload, ensure you use FormData for multipart requests.

const formData = new FormData();
formData.append('name', 'New User');
formData.append('avatar', fileInput.files[0]);

fetch('/api/users', {
    method: 'POST',
    body: formData
    // Note: Don't set Content-Type header manually, 
    // fetch will set it for multipart/form-data with boundary automatically.
}).then(res => res.json());
